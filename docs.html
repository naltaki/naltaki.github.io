Setup

  • Clone the project.
  • Create a Parse.com app.
  • Execute this command to create a Parse class "Events" with your own credentials.
    • curl -X POST \ -H "X-Parse-Application-Id: 123456" \ -H "X-Parse-Master-Key: a1b2c3d4e5f6" \ -H "Content-Type: application/json" \ -d ' { "className": "Events", "fields": { "ACL":{ "type":"ACL" }, "createdAt":{ "type":"Date" }, "description":{ "type":"String" }, "guests":{ "type":"Array" }, "objectId":{ "type":"String" }, "rsvpLimit":{ "type":"Number" }, "startAt":{ "type":"Date" }, "title":{ "type":"String" }, "updatedAt":{ "type":"Date" }, "venueAddress":{ "type":"String" }, "venueGeoLocation":{ "type":"GeoPoint" }, "venueLink":{ "type":"String" }, "venueName":{ "type":"String" } } }' \ https://api.parse.com/1/schemas/Events
  • Change default configuration values in index.html.
  • Push the code to GitHub Pages.
  • GitHub pages is hosting the app.
  • Parse.com is managing the back end.
